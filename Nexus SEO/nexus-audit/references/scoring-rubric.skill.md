# scoring-rubric.skill.md

**Role:** Grading rubrics for all 7 audit dimensions. Always load alongside `blog-post-auditor.skill.md`. Never score without loading this file.

---

## Search Intent Match

| Grade | Criteria |
|---|---|
| A (Match) | Blog format matches the dominant SERP format exactly. If top 3 are tutorials, this is a tutorial. Same depth level. |
| B (Partial) | Blog format is close but with a detectable difference. E.g., top 3 are pure tutorials, this is tutorial + product pitch. |
| C (Weak) | Blog format is in the right category but wrong emphasis. E.g., top 3 focus on workflow, this focuses on features. |
| D (Mismatch) | Blog format conflicts with SERP intent. E.g., top 3 are comparison pages, this is a deep-dive. |
| F (Wrong) | Blog serves a fundamentally different intent. E.g., top 3 are official docs, this is a marketing page. |

---

## Content Depth vs Competition

| Grade | Criteria |
|---|---|
| A (Exceeds) | Word count ≥ top 3 average. Code examples ≥ top 3. Tables ≥ top 3. Covers all subtopics they cover PLUS at least 1 unique section. |
| B (Matches) | Word count within 20% of top 3 average. Similar code example count. No major subtopic gaps. |
| C (Below) | Word count 50–80% of top 3 average. Missing 1-2 subtopics competitors cover. Fewer code examples. |
| D (Significantly Below) | Word count < 50% of top 3. Missing 3+ subtopics. Visibly thinner than competitors. |
| F (Inadequate) | Substantially shorter and shallower than all top 3 results. Would never compete on content alone. |

## E-E-A-T Signals

| Grade | Criteria |
|---|---|
| A (Strong) | Author bio with specific experience. Original data or benchmarks. Multiple real screenshots/outputs. First-person experience throughout. |
| B (Adequate) | Author bio present. Some original framing. A few proof points. Occasional first-person voice. |
| C (Weak) | Generic author bio. No original data. Screenshots are stock or generic. No first-person signals. |
| D (Very Weak) | No author bio or generic "content team." No proof points. Reads like compiled research. |
| F (Absent) | No author attribution. No originality signals. Could have been generated by any AI from training data. |

## Technical SEO

| Grade | Criteria |
|---|---|
| A (Clean) | Perfect heading hierarchy. 10+ contextual internal links. Schema markup present. Clean URL. Fast page. |
| B (Minor Issues) | 1-2 heading inconsistencies. 5-9 internal links. URL is fine. Minor image optimization gaps. |
| C (Notable Issues) | Multiple heading problems. Fewer than 5 internal links. No schema. Some forced links. |
| D (Major Issues) | Broken hierarchy (orphan H3s, missing H2s). 0-2 internal links. Heavy images. URL issues. |
| F (Severe) | No heading structure. No internal links. Broken elements. URL not keyword-relevant. |

## AEO Dimensions (each scored 1–5)

### Direct Answer Blocks

| Score | Criteria |
|---|---|
| 5 | Every H2 section opens with a direct, extractable answer statement. An LLM could cite any section independently. |
| 4 | Most sections open with direct answers. 1-2 sections bury the answer in prose. |
| 3 | Some answer blocks present but inconsistent. Key answers are in paragraphs 2-3 of sections. |
| 2 | Few direct answers. Most sections build up to the point instead of stating it first. |
| 1 | No extractable answer blocks. Content is conversational without structured claim statements. |

### Structured Data

| Score | Criteria |
|---|---|
| 5 | 3+ comparison tables, numbered processes, and decision frameworks. All with specific data. |
| 4 | 2 tables + some numbered lists. Most comparisons are structured. |
| 3 | 1 table or comparison matrix. Some structured elements but mostly prose. |
| 2 | Lists present but no tables or comparison matrices. |
| 1 | Pure prose. No structured data elements an LLM could parse. |

### Unique Attribution Signals

| Score | Criteria |
|---|---|
| 5 | Multiple brand-specific claims ("[brand.brand_name]'s analysis shows...", "According to our data..."). Original frameworks named. Brand name reads from active brand file field: brand.brand_name |
| 4 | Some branded claims. At least 1 unique framing or named framework. |
| 3 | Generic expertise signals but nothing uniquely attributable to the brand. |
| 2 | Could be written by anyone. No brand-specific voice or data. |
| 1 | Pure commodity content. Zero attribution signals. |

### Freshness Signals

| Score | Criteria |
|---|---|
| 5 | Publication date visible. Tool versions mentioned (e.g., "Playwright 1.58"). Current pricing/features. |
| 4 | Date visible. Most version info current. 1-2 references slightly dated. |
| 3 | Date visible but no version pinning. Could be from any year. |
| 2 | No date visible. Some outdated information. |
| 1 | No date. Outdated tool versions. Stale comparisons. |

### Definition Sentences

| Score | Criteria |
|---|---|
| 5 | Every key concept has a clean, 1-sentence definition at the start of its section. LLM-extractable. |
| 4 | Most concepts defined. Definitions mostly at section start. |
| 3 | Some definitions present but buried in prose or inconsistent. |
| 2 | Few definitions. Key terms assumed known without explanation. |
| 1 | No explicit definitions for any concept. |

### FAQ Section

| Score | Criteria |
|---|---|
| 5 | 5-8 questions matching real LLM query patterns. Direct answer first sentence. Inline links. |
| 4 | FAQ present with good questions. Some answers could be more direct. |
| 3 | FAQ present but questions are surface-level or don't match search patterns. |
| 2 | 1-3 questions in a minimal FAQ. Low quality. |
| 1 | No FAQ section. |

## Heading Quality

Score as: (number of H2s passing all 3 tests) / (total H2 headings)

The 3 tests every H2 must pass:
1. **Keyword Test:** Contains the target keyword or semantic variant
2. **Answerability Test:** Can be rephrased as a question an LLM would answer with this section
3. **Tension Test:** Creates curiosity, an information gap, or a challenge

| Grade | Criteria |
|---|---|
| A | 90%+ H2s pass all 3 tests. No generic headings. |
| B | 75–89% pass all 3. 1 generic heading remains. |
| C | 60–74% pass all 3. Multiple generic headings. |
| D | 40–59% pass. Most headings are functional but not optimized. |
| F | Below 40%. Headings are generic or vague throughout. |

---

## Catch-22 Assessment

Binary: **Pass** or **Fail**.

**Pass if ALL of these are true:**
- The article contains at least 2 things a reader cannot get by prompting an LLM
- There is at least 1 bookmarkable asset (table, template, prompt library, cost data)
- The target audience would choose this blog over prompting Claude because of specific, identifiable value

**Fail if ANY of these are true:**
- The entire article could be reproduced by a well-prompted LLM
- There are no original artifacts, data, or community-synthesized insights
- A senior developer would close the tab after scanning the headings

## Overall Grade Conversion

Applies to all letter-graded dimensions:

| Grade | Score Range | Meaning |
|---|---|---|
| A | 90–100% | Competitive or better than top 3 |
| B | 75–89% | Solid, minor fixes needed |
| C | 60–74% | Ranking is possible but at a disadvantage |
| D | 40–59% | Significant work needed to compete |
| F | 0–39% | Not competitive; major rebuild required |
